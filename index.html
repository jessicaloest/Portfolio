XING Webseite

XING Suche

Die XING Suche sollte mithilfe des neuen XING-Designs neu bzw. Userfreundlicher gebaut werden. Dabei habe ich den jetzigen Aufbau der XING Suche behalten und die das alte Design in das neue Design geändert.

XING alle Plattformen

XING Suche mit einer Person

Es sollte die Suche mir einem echten User nachgebaut werden. Dabei sollen alle verschiedenen Unterkategorien auf einer Seite angezeigt weden. "Top Suchergebnisse" und "Weitere Suchergebnisse" sollen die einzigen Kategorien sein.

XING Webseite

XING Empty Search

Es sollte ein Empty Search gebaut werden, welcher dem User mehr Inteaktion mit der Suche ermöglichen soll, wenn dieser nicht mehr weiß wonach er sucht.

XING Webseite

Icons

Icons für Premium-Trophäen, die der Nutzer belohnen sollen, wenn dieser z.B. sehr aktiv auf de Plattform ist.

Schule

Logo und Visitenkarte

Es sollte ein Weinlogo , Mockup und eine Visitenkate für Sauewein erstellt werden. Das Logo sollte zeittlos, seriös, hochwertig und bodentädig wirken.

Notification Center

Empty State

Das Notification Center ist eine React-Applikation die in der XING Webseite lebt. Sie ist zum Teil abhängig von den Daten, die sie von der XING Webseite bekommt. Es sollte ein Empty State für das Notifications Center zu sehen sein, wenn der Nutzer noch keine Mitteilungen hat. Um den Empty State darzustellen habe ich eine React Komponente gebaut, die aus einem Bild, einer Headline, einem Text und einen Button besteht.

System E-Mails

Neue Information

Ich habe bei einigen E-Mails neue Information zum Beispiel bei einem anstehenden Event hinzugefügt. Der Eventveranstalter, freie Plätze und Eventort sollten bei den Events hinzugefügt werden. Vorhandene Information sollte angezeigt werden. Im Fall, dass die Information nicht vorhanden ist, wird ein Ersatz oder eines leeres Feld angezeigt.

System E-Mails

Neues Design

Die E-Mails werden in der neuen Designsprache der XING Webseite angepasst. In den neuen E-Mails werden Komponenten zusammengefügt und mit unechten Daten gefüllt. Dabei werden schon vorhandene Komponenten verwendet oder neu erstellt. Jede neu erstellte E-Mail wird sowohl als HTML Template Variante und Textvariante erstellt.
